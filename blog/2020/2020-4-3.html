有A, B两个简单的数组，内部不包含对象，如何确定两个数组的内容是否一样？

我能想到的简单粗暴的方法是直接A.toString() === B.toString()。刚刚闲的蛋疼看Vue的源代码，里面有个写法，我很喜欢，也许这只是很正常的操作，只不过我太菜少见多怪而已。

这只是一个函数内部的两行，我包一个函数便于理解：

function equal(A, B) {
    return A.length === B.length && A.every(function (n, i) {
        return n === B[i]
    })
}

它看起来很简单，要是我写，也能实现同样的功能，但就不如人家这个好看。

今天看几条新闻，关于瑞幸咖啡股价暴跌的信息。我突然想起前一段时间在网上看到说有国外公司做空瑞幸股价的事情，我在手机上查了一些，大概就是两个月以前，国外有家叫‘浑水公司’的公司，公布了一个匿名的调查报告，揭露了瑞幸咖啡的种种不实经营行为。话说浑水公司为了做空瑞幸的股价，可是费了很大的功夫，调查花费的精力与手段专业都让我有点震惊。

不过在看到这些信息的时候，我甚至不知道做空要怎么挣钱，所以又在网上查查。我大概了解的意思是：做空公司先在要做空的公司股票高价位的时候从券商那里借来股票，然后把股票卖掉，再揭露该公司的不良经营的信息，让这家公司的股票下跌，然后再花较低的价格买入该公司的股票还给券商。高价位的卖出，和低价位的买入，中间的差价就是做空公司的利润。

从券商那里“借”只是一种简单的说法，事实上应该是一种正常的商业模式，因为做空公司肯定要付手续费给券商。比如期货，都是资本市场的游戏，只不过我不懂。

股票盈利的方式，用非常简单的话讲，就是：低价买入、高价卖出。我非常敬佩用做空方式盈利的人，因为，只要依靠股票价格的差值挣钱，就不可能脱离刚刚说的原则。

做空的人非常巧妙的把这个原则调换了位置，原本是先低价买入、后高价卖出，现在变成了先高价卖出，后低价买入。

原则变了，还能不能盈利？还能不能操作？就像刚刚上面的简单例子一样，我是绝对想不出这种骚操作。高价卖出、低价买入是可以操作的，但中间需要很多流程去处理，你需要利用现行的商业模式，再利用自己可以操作的东西，创造出对自己最有利的条件，然后一步一步实现自己的愿望。说实话，我觉得这样的盈利方式简直是魔法。

我的大脑是极其呆板的，我觉得我经常陷入各种非常失败的思维模式里，对于偶尔能一撇别人思想的闪光点时，也只能叹为观止，在边上喊666了。
